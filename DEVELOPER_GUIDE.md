## 🚀 Quick Start Commands

### Development (Windows)

```powershell
# All-in-one launcher: Starts API server + Electron app (RECOMMENDED)
.\start-nebula.ps1

# Or use npm script:
npm run start:vpn

# Or manually (separate terminals):
# Terminal 1: API Server
cd server
npm start

# Terminal 2: Electron app (Run as Administrator!)
.\start-vpn-admin.ps1
```

### Verify Installation

```powershell
# Check DNS enforcement is working
.\verify-dns-simple.ps1
# OR:
npm run verify:dns

# Run security audit
npm run security:audit

# Encrypt existing database (if upgrading)
npm run db:migrate:encrypt
```

## 🔐 Security Features

### 1. Database Encryption (AES-256-GCM)

**Location**: `server/src/db.js`

**Encrypted Fields**:
- `twoFactorSecret` - TOTP secrets
- `twoFactorTempSecret` - Temporary 2FA setup tokens
- `oauthId` - OAuth provider identifiers

**How It Works**:
```javascript
// Automatic encryption on write
users.set('user@example.com', {
  email: 'user@example.com',
  twoFactorSecret: 'PLAINTEXT_SECRET'  // ← Encrypted before DB write
});

// Automatic decryption on read
const user = users.get('user@example.com');
console.log(user.twoFactorSecret);  // ← Returns decrypted: "PLAINTEXT_SECRET"
```

**Key Management**:
- Key stored in `ENCRYPTION_KEY` env var (64 hex characters)
- Generated by `generate-secrets.js`
- **Never change the key in production** (breaks existing data)
- For key rotation, see `DATABASE_ENCRYPTION.md`

**Migration**:
```bash
cd server
node src/migrations/encrypt-existing-data.js
```

---

### 2. DNS Enforcement (OS-Level)

**Location**: `electron/vpn-tunnel.js` → `_dnsWindows()`, `setDNS()`

**How It Works**:
1. VPN connects and gets IP (e.g., `10.8.0.2`)
2. `setDNS(['1.1.1.1', '1.0.0.1'])` called
3. Executes: `netsh interface ipv4 set dnsservers name="Nebulavpn" source=static address=1.1.1.1`
4. Windows network stack updated
5. All DNS queries → 1.1.1.1 (Cloudflare)

**Requirements**:
- **Windows**: Administrator privileges
- **Linux**: sudo/root access
- **macOS**: sudo/root access

**Error Handling**:
```javascript
try {
  await tunnel.setDNS(['1.1.1.1', '1.0.0.1']);
} catch (error) {
  if (error.code === 'REQUIRES_ADMIN') {
    // Show UI prompt to restart as admin
  }
}
```

**Verification**:
```powershell
.\verify-dns-simple.ps1
# Should show:
# [PASS] DNS Configured: 1.1.1.1, 1.0.0.1
# [PASS] DNS Working: Using VPN DNS server!
```

### 3. Administrator Privilege Detection

**Location**: `electron/main.js` → `isRunningAsAdmin()`, `promptForElevation()`

**How It Works**:
```javascript
// On app.whenReady()
hasAdminPrivileges = await isRunningAsAdmin();

if (!hasAdminPrivileges) {
  console.warn('⚠️  NOT running as Administrator - DNS enforcement will FAIL');
  
  // Prompt user to restart (non-blocking, delayed 2 seconds)
  setTimeout(async () => {
    await promptForElevation(mainWindow);
  }, 2000);
}
```

**IPC Handler** (renderer can query status):
```javascript
// Renderer (React)
const { hasAdmin } = await window.electron.invoke('check-admin-privileges');

if (!hasAdmin) {
  // Show warning banner in UI
}
```

## 🔧 Development Workflows

### Adding a New API Route

1. Create route file: `server/src/routes/myroute.js`
2. Add authentication middleware if needed
3. Register in `server/src/index.js`:
   ```javascript
   const myRoute = require('./routes/myroute');
   app.use('/api/myroute', myRoute);
   ```

### Adding a New Electron IPC Handler

1. Register in `electron/main.js` inside `app.whenReady()`:
   ```javascript
   ipcMain.handle('my-handler', async (event, args) => {
     // Your logic
     return { success: true, data: 'result' };
   });
   ```

2. Use in renderer via `window.electron.invoke()`:
   ```javascript
   const result = await window.electron.invoke('my-handler', { foo: 'bar' });
   ```

### Adding Encrypted Database Fields

1. Update `ENCRYPTED_FIELDS` array in `server/src/db.js`:
   ```javascript
   const ENCRYPTED_FIELDS = [
     'twoFactorSecret', 
     'twoFactorTempSecret', 
     'oauthId',
     'myNewSensitiveField'  // ← Add here
   ];
   ```

2. Add to `TO_DB` mapping if it's a new column:
   ```javascript
   const TO_DB = {
     // ...existing fields...
     myNewSensitiveField: 'my_new_sensitive_field'
   };
   ```

3. Create migration to add column (if needed):
   ```javascript
   db.exec(`ALTER TABLE users ADD COLUMN my_new_sensitive_field TEXT`);
   ```

4. Encryption/decryption happens automatically!

## 📝 Environment Variables Reference

### Server (`server/.env`)

```bash
# ── Server Config ──────────────────────────────────────
PORT=3001
NODE_ENV=development
HOST=0.0.0.0

# ── Security Keys ──────────────────────────────────────
JWT_SECRET=<128 hex chars>                    # Generate with generate-secrets.js
JWT_REFRESH_SECRET=<128 hex chars>            # Generate with generate-secrets.js
ENCRYPTION_KEY=<64 hex chars>                  # For AES-256-GCM database encryption
JWT_EXPIRES_IN=7d

# ── Admin Account ──────────────────────────────────────
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=<strong password>

# ── WireGuard Config ───────────────────────────────────
WG_INTERFACE=wg0
WG_SERVER_PUBLIC_KEY=<base64 44 chars>        # cat /etc/wireguard/server_public.key
WG_SERVER_ENDPOINT=<ip_or_domain>:51820       # Your VPS IP
WG_DNS=1.1.1.1,1.0.0.1                        # DNS servers for VPN
WG_SUBNET=10.8.0.0/24

# ── Rate Limiting ──────────────────────────────────────
RATE_LIMIT_MAX=1000                            # Max requests per window
RATE_LIMIT_WINDOW_MS=900000                    # 15 minutes

# ── CORS ───────────────────────────────────────────────
ALLOWED_ORIGINS=http://localhost:3000,https://yourdomain.com

# ── Development ────────────────────────────────────────
ALLOW_INSECURE_WG_DEV=true                     # Allow VPN without real WireGuard
LOG_LEVEL=info
```

## 🐛 Common Issues & Solutions

### DNS Enforcement Not Working

**Symptoms**: VPN connects but DNS still uses ISP (`64.71.x.x` instead of `1.1.1.1`)

**Causes**:
1. Not running as Administrator
2. WireGuard adapter has no DNS servers

**Solutions**:
```powershell
# 1. Restart as admin
.\start-vpn-admin.ps1

# 2. Verify DNS enforcement
.\verify-dns-simple.ps1

# 3. Check DNS on adapter (as admin)
Get-DnsClientServerAddress -InterfaceAlias Nebulavpn

# 4. Manually set DNS (as admin)
netsh interface ipv4 set dnsservers name="Nebulavpn" source=static address="1.1.1.1"
```

---

### VPN Gets APIPA Address (169.254.x.x)

**Symptoms**: VPN adapter shows `169.254.175.132` instead of `10.8.0.x`

**Causes**:
1. WireGuard server unreachable
2. Invalid WireGuard config
3. Firewall blocking UDP 51820

**Solutions**:
```powershell
# 1. Test server connectivity
Test-NetConnection -ComputerName <your-server-ip> -Port 51820

# 2. Check WireGuard config
wg showconf Nebulavpn  # (as admin)

# 3. Verify server is running
# SSH to server:
sudo wg show wg0
```

---

### Database Encryption Errors

**Symptoms**: `ENCRYPTION_KEY must be a 64-character hex string`

**Cause**: Missing or invalid `ENCRYPTION_KEY` in `server/.env`

**Solution**:
```bash
# Generate new secrets (includes encryption key)
node scripts/generate-secrets.js \
  --admin-email your@email.com \
  --admin-password "YourPassword123!"

# Or manually generate encryption key
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
# Copy output to ENCRYPTION_KEY in server/.env
```
